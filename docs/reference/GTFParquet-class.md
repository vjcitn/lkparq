# GTFParquet class

<div class="ref-description section level2">

An S4 class for accessing GTF annotations stored in Parquet format.
Unlike TxDb, preserves all GTF attributes (gene\_type, gene\_name,
transcript\_support\_level, tags, etc.)

</div>

<div class="section level2">

## Usage

<div class="sourceCode">

``` r
# S4 method for class 'GTFParquet'
genome(x)

# S4 method for class 'GTFParquet'
seqinfo(x)
```

</div>

</div>

<div class="section level2">

## Arguments

-   x:

    A `GTFParquet` object.

</div>

<div class="section level2">

## Value

A [Seqinfo](https://rdrr.io/pkg/Seqinfo/man/Seqinfo-class.html) object
containing chromosome names and genome build.

</div>

<div class="section level2">

## Details

GTFParquet objects are created by the `GTFParquet` constructor function
from a directory of Parquet files generated by `gtf_to_parquet.py`.

The class implements methods for GenomicFeatures generics including
`genes`, `transcripts`, `exons`, `cds`, `exonsBy`, `cdsBy`, and
`transcriptsBy`.

All methods support a `filter` argument for efficient querying (e.g.,
`filter = list(gene_type = "protein_coding")`).

</div>

<div class="section level2">

## Slots

-   `path`:

    Character. Path to the Parquet directory.

-   `files`:

    List. Paths to individual Parquet files.

-   `available`:

    Logical vector. Which files are present.

-   `is_partitioned`:

    Logical. Whether genes are partitioned by chromosome.

-   `.genome`:

    Character. Reference genome build (e.g., "GRCh38").

</div>

## Examples

<div class="sourceCode">

``` r
if (FALSE) { # \dontrun{
# Create from Parquet directory
gtf <- GTFParquet(system.file("gc49", package="lkparq"))

# Extract genes with full attributes
gr <- genes(gtf)
mcols(gr)  # gene_name, gene_type, level, tags, etc.

# Filter by gene type
pc <- genes(gtf, filter = list(gene_type = "protein_coding"))
} # }
```
